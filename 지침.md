# 🚗 [Prompt] MotorWatch 예지보전 대시보드 V3 스트림릿(Streamlit) 최종 전환 및 ML 통합 지침

제공된 `MotorWatch Dashboard v3 _standalone_.html` 파일의 UI/UX 디자인 사양(다크 모드, 고대비 레이아웃, 컬러 시스템)을 토씨 하나 틀리지 않고 완벽하게 재현하면서, 백엔드 ML 모델 예측 및 실시간 SHAP XAI 연동이 가능한 단일 파이썬 스트림릿(`app.py`) 코드를 작성해줘.

## 1. 데이터 정합성 및 백엔드 파이프라인 (Data & ML Pipeline)
- **사용 변수 (6개 정형 피처):** UI에서 가상의 물리량(RPM, 토크 등)은 배제하고 실제 데이터 구조인 3대 센서의 통계적 특징만 다룬다.
  - `Current_U_mean`, `Current_U_var` (모터 U상 전류)
  - `Vib_Motor_mean`, `Vib_Motor_var` (모터 진동)
  - `Vib_TM_mean`, `Vib_TM_var` (감속기 진동)
- **다중 클래스 레이블 (5대 Target Class):** `NORMAL`, `ECC10`, `ECC20`, `DEMAG`, `REDUC`
- **ML 및 SHAP 로드 뼈대 구현:** 실제 학습된 가중치 모델 파일(`.pkl`)과 `shap.TreeExplainer`를 스크립트 상단에서 `@st.cache_resource`로 안전하게 로드할 수 있는 코드 주석 및 구조를 포함해줘.

## 2. UI/UX 디자인 정합성 제약 (Design Specifications)
- 스트림릿의 기본 테마(흰색 배경 또는 투박한 컴포넌트)를 사용하지 말고, `st.markdown`과 커스텀 CSS 지입을 통해 HTML 원본의 세련된 다크 모드 톤앤매너를 100% 모사할 것.
  - **전체 배경색:** `#0F172A` (Slate Dark)
  - **컴포넌트 카드 배경색:** `#1E293B` 로 통일하고 테두리 선은 `#334155` 처리.
  - **폰트:** 프리미엄 인포테인먼트 스타일의 'Inter' 또는 시스템 산세리프 폰트 적용.

## 3. 핵심 화면 구성 및 인터랙션 요구사항

### 3.1 상단 4구 종합 KPI 카드
- HTML과 동일한 여백과 그리드로 아래 4가지 지표를 배치해줘:
  1. **Motor Health Score:** 0~100점 형태 (예: 76 / 100 점)
  2. **Primary Diagnosis:** 현재 가장 확률이 높은 대표 상태 (예: ECC10)
  3. **Degradation Level:** [ 정상 ➔ 주의 ➔ 결함 ] 계층형 텍스트 및 상태에 맞는 컬러(초록/주황/빨강) 동적 하이라이트.
  4. **System Latency:** 초경량 ML 모델의 연산 속도 고정 표시 (1.2 ms)

### 3.2 메인 영역: 다중 고장 예측 및 실시간 SHAP 워터폴 클릭 연동 (좌우 분할)
- **[우측] 다중 고장 유형 확률 모니터:**
  - 5가지 레이블의 예측 확률(%)을 세로형 프로그레스 바 리스트로 동시에 띄운다.
  - 스트림릿의 기본 `st.progress` 대신 HTML과 똑같은 얇고 세련된 둥근 모서리의 커스텀 프로그레스 바(Progress Bar)를 구현할 것.
  - **인터랙션:** 사용자가 각 고장 유형 항목을 마우스로 클릭할 수 있는 인터랙티브 구조(`st.radio` 또는 커스텀 버튼)를 설계하고, 사용자가 선택한 항목은 UI 상에서 테두리 하이라이트 효과가 들어가게 해줘.
- **[좌측] 선택된 고장별 실시간 SHAP 워터폴 차트:**
  - 우측 리스트에서 사용자가 특정 고장을 클릭하면, **좌측의 워터폴 차트가 해당 고장 확률에 기여한 분석 화면으로 즉시 스위칭(업데이트)**되어야 한다.
  - Matplotlib 또는 Plotly를 활용해 다크 모드 배경과 완벽히 정합되는 고해상도 워터폴 차트를 구현할 것.
  - 정상 기준점(Baseline)에서 시작해 각 센서 변수가 확률을 위로 끌어올렸으면 빨간색(`+`), 낮췄으면 초록색/파란색(`-`) 계단식 수직 막대로 가독성 있게 표현해줘.

### 3.3 하단 영역: 실시간 관제 시스템 로그
- 실제 제어실 터미널 화면처럼 타임스탬프와 상태 등급 태그가 찍힌 스크롤 가능한 형태의 '실시간 시스템 이벤트 로그(Alert Log)'창을 디자인해줘.

## 4. 시연 편의 기능 (Streaming Control)
- 대시보드 상단에 수동으로 센서 분산 수치를 조절할 수 있는 슬라이더(`st.slider`) 패널을 접이식(`st.expander`)으로 포함하고, 버튼 하나로 데이터가 자동 요동치며 리프레시되는 **[실시간 데이터 스트리밍 토글 버튼]** 로직을 세션 상태(`st.session_state`)와 `st.rerun()`을 이용하여 완벽하게 구현해줘.
